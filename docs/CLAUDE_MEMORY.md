# Auto Memory — AIAdminka Project

## Шорткаты пользователя
- **"йцу"** = "Согласно CLAUDE.md" — пользователь напоминает что нужно действовать по инструкциям CLAUDE.md

## Критически важно (читай ПЕРВЫМ при каждой задаче)
- **ВСЕГДА сверяться с CLAUDE.md** перед началом любой задачи
- При реализации фичи — строго следовать рабочему процессу:
  architect -> код -> tester -> code-reviewer -> security-reviewer -> docs -> DoD
- Не пропускать шаги даже в длинных диалогах
- Пользователь просил: всегда применять CLAUDE.md, не ждать явного напоминания
- **ПРОАКТИВНО вызывать скиллы** — НЕ ждать пока пользователь вызовет сам

## Протокол сохранения контекста
Когда диалог становится длинным — **сохрани чекпоинт ДО потери контекста**:
1. Записать в `memory/checkpoint.md`: задачу, прогресс, решения, оставшиеся шаги
2. Сказать пользователю: "Контекст на исходе, сохранил чекпоинт — пересоздай сессию"
3. В новой сессии: прочитать `memory/checkpoint.md` и продолжить

При получении сообщения "продолжи" — СРАЗУ читай `memory/checkpoint.md`.
Подробнее: см. `memory/context-protocol.md`

## Скиллы — вызывать автоматически
- `/architect` — перед реализацией нетривиальной фичи
- `/test` — после написания кода
- `/review` — после тестов
- `/security` — если код работает с вводом/auth/API/данными
- `/simplify` — после ревью, если есть что упростить
- `/docs` — когда изменения значимые
- `/release-notes` — при выпуске новой версии
- `/research` — для объёмных исследовательских задач (Gemini)
- `/diagram` — при проектировании архитектуры
- `/status` — по запросу или в начале сессии
