# Документация SimpleAIAdminka

## Как обновить историю обновлений

При добавлении новых функций или исправлений нужно обновить **ТРИ** места:

### 1. Основной файл данных
**Файл:** `data/changelog.json`

Добавьте новую версию в начало массива `updates`:

```json
{
  "version": "1.X.X",
  "updates": [
    {
      "version": "1.X.X",
      "date": "2025-MM-DD",
      "title": "Название обновления",
      "changes": [
        {
          "category": "Новое",
          "items": [
            "Описание новой функции"
          ]
        }
      ]
    }
  ]
}
```

### 2. Fallback данные в JS
**Файл:** `js/changelog.js`

Обновите `fallbackData` (строка ~16):
- Измените версию
- Добавьте новое обновление в начало массива updates
- **ВАЖНО:** Это резервные данные для работы без сервера

### 3. Версия в HTML
**Файл:** `index.html`

Обновите номер версии (строка ~23):
```html
<span>Версия: 1.X.X</span>
```

## Категории изменений

Используйте следующие категории:

- **Новое** - новые функции и возможности
- **Исправлено** - исправления багов
- **Улучшено** - улучшения существующих функций
- **Удалено** - удаленные функции
- **Изменено** - изменения в поведении
- **Интерфейс** - изменения UI/UX
- **Архитектура** - изменения в структуре кода

## Проверка

После обновления проверьте:

1. ✅ Откройте документацию в браузере
2. ✅ Нажмите кнопку "История обновлений"
3. ✅ Убедитесь, что новая версия отображается первой с меткой "Latest"
4. ✅ Проверьте, что номер версии в заголовке совпадает с последней версией

## Структура файлов

```
documentation/
├── index.html           # Главная страница документации
├── README.md           # Этот файл
├── data/
│   └── changelog.json  # Основные данные истории обновлений
├── js/
│   └── changelog.js    # Логика отображения + fallback данные
└── css/
    └── documentation.css
```

## Советы

- Сохраняйте консистентный стиль описаний
- Группируйте связанные изменения вместе
- Используйте конкретные глаголы (добавлено, исправлено, улучшено)
- Указывайте названия модулей в скобках, если нужно
- Для больших обновлений добавляйте подпункты с отступами

## Пример полного обновления

1. Обновите `data/changelog.json` - добавьте новую версию
2. Обновите `js/changelog.js` - синхронизируйте fallbackData
3. Обновите `index.html` - измените номер версии
4. Откройте страницу и проверьте отображение

Готово! История обновлений синхронизирована.
