Быстрый старт

AIAdminka — веб-приложение для управления партнёрской программой: партнёры, команда, расчёт трафика, Excel-отчёты, роли и доступы. Данные хранятся в облаке Google Drive, вход через Google OAuth, изменения синхронизируются между вкладками в реальном времени.

Состоит из 9 модулей. Доступные пункты меню зависят от вашей роли — то, на что у роли нет прав, в меню просто не отображается.

Как войти

  1. Откройте приложение в браузере.
  2. Нажмите «Войти через Google».
  3. Выберите свой Google-аккаунт и подтвердите доступ. После входа откроется главная страница (или экран ожидания приглашения, если доступ ещё не выдан).

Навигация

  • Слева — боковое меню (sidebar): клик по пункту открывает нужный модуль.
  • Пункты, недоступные вашей роли, скрыты автоматически.

Документация

Эта страница: справочник и история обновлений

Что встретится во всех модулях

  • При открытии страницы виден полноэкранный спиннер «Загрузка...» — пока подгружаются данные.
  • Кнопки действий скрыты, если у вашей роли нет соответствующего права.
  • Изменения применяются к экрану сразу (оптимистичный UI); при ошибке — автоматический откат и красное уведомление.
  • Результаты операций показываются всплывающими уведомлениями (Toast) в правом углу.
  • Опасные действия (удаление, выход) требуют подтверждения в модальном окне.

Админ-панель

URL: /admin/  •  Доступ: только admin

Назначение

Центр управления системой: команды, пользователи, запросы на регистрацию, роли и права, история действий. Состоит из 5 вкладок: Команды, Пользователи, Запросы, Роли и права, История.

Как это сделать

  • Создать команду: вкладка «Команды» → «Создать команду» → в окне заполнить Название, Руководитель (email), Описание → «Создать».
  • Редактировать/удалить команду: вкладка «Команды» → клик по карточке команды → в окне «Редактирование команды» изменить название/описание/статус → «Сохранить»; для удаления — «Удалить» → подтвердить (окно показывает имя команды и число участников).
  • Изменить пользователя: вкладка «Пользователи» → найти через поиск или фильтры (Все команды / Все роли / Все статусы) → клик по строке → выбрать Команду, Роль, Статус → «Сохранить». Удаление — кнопка «Удалить» (себя удалить нельзя).
  • Обработать запрос на регистрацию: вкладка «Запросы» → карточка запроса → одобрить или отклонить (отклонение подтверждается в окне с именем заявителя).
  • Управлять ролями: вкладка «Роли и права» → суб-вкладка «Управление ролями» → «+ Добавить роль» (название, цвет) → «Сохранить»; «Сбросить» откатывает имена; удаление роли — с переназначением её пользователей.
  • Настроить права: суб-вкладка «Настройка прав» → матрица роль × модуль (Просмотр / Редактирование / Удаление по модулям) → «Сохранить». Руководители смогут назначать только разрешённые здесь роли.
  • Посмотреть историю: вкладка «История» → фильтры по дате (С / По), поиск, фильтр действий → «Загрузить ещё» для пагинации.

Модальные окна закрываются крестиком в шапке окна.

Кто может

Только admin. Остальные роли модуль в меню не видят.

Партнёры

URL: /partners/  •  Доступ: все авторизованные роли (по матрице прав)

Назначение

Список партнёров (субагентов) с методами работы, статусами, кастомными полями и счётчиками (DEP / WITH / COMP). Поддерживает импорт и экспорт.

Как это сделать

  • Добавить партнёра: кнопка «Добавить» в шапке таблицы → в форме справа заполнить Субагент, ID Субагента, выбрать Метод и Шаблон полей, заполнить счётчики DEP/WITH/COMP, при необходимости загрузить фото (клик по аватару → обрезка) → «Сохранить».
  • Изменить статус: клик по строке партнёра → в карточке справа dropdown статуса → «Открыт» (зелёный) или «Закрыт» (красный).
  • Редактировать/удалить: клик по строке → карточка справа → «Редактировать» (откроет форму) или «Удалить».
  • Найти/отсортировать: поиск «по субагенту, методу…» в шапке; клик по заголовкам колонок Метод / Субагент / Статус — сортировка.
  • Настроить колонки: иконка-шестерёнка «Настройка колонок» → отметить нужные колонки.
  • Экспорт: «Экспорт списка» → выбрать JSON или Excel (для Excel — шаблон колонок) → «Экспортировать».
  • Импорт: «Импорт списка» → JSON (выбрать или перетащить файл) ИЛИ Excel (шаг 1: выбрать шаблон → шаг 2: скачать пример структуры, загрузить .xlsx/.xls) → «Импортировать».

Кто может

Видимость задаётся матрицей прав admin. Редактирование/удаление — роли с правом edit на «Партнёры» (например partners_mgr, leader); просмотр — роли с правом view.

Заведение партнёра

URL: /partner-onboarding/  •  Доступ: sales (executor) + admin/leader (reviewer)

Назначение

Пошаговый процесс заведения партнёра с разделением ролей: executor заполняет шаг, reviewer проверяет. Заявка проходит 8 шагов от лида до активации: Входящий лид → Полная информация → Создание аккаунта → Заполнение профиля → Антифрод проверка → Пополнение счёта → Корп. мессенджер → Финализация карточки. Статусы заявки: Новая, В работе, На проверке, Проверка пройдена, Требуется доработка, Завершена, Отменена.

Как это сделать

  • Создать заявку: «Новая заявка» → выбрать источник лида (Telegram / Баннер / Реферал / Холодный звонок / Конференция / Сайт / Другое) → заполнить начальные поля.
  • Заполнить и отправить шаг (executor): открыть заявку → заполнить поля текущего шага (сохраняются автоматически по мере ввода) → «На проверку» (если у шага есть reviewer) или «Далее» (если нет).
  • Проверить шаг (reviewer): открыть заявку на проверке → «Одобрить» (переход к следующему шагу) или «Отклонить» с комментарием (заявка → «Требуется доработка», возврат executor).
  • Передать заявку: reassign другому исполнителю с причиной (Отпуск / Больничный / Увольнение / Перераспределение) и комментарием.
  • Фильтровать заявки: dropdown «Все заявки» → по статусу; поиск; чекбоксы для массового удаления выбранных.
  • Настройки (admin/leader): dropdown «Настройки» → Источники (импорт лидов из Google Sheets) / Условия / Роли по шагам (per-step executor и reviewer).
  • Admin-операции: откат шага (rollback), отмена (cancel), восстановление (reactivate).

Кто может

  • sales — создаёт и заполняет заявки (executor), видит свои заявки.
  • antifraud — reviewer на шаге антифрод-проверки.
  • admin/leader — reviewer + настройки (Источники / Условия / Роли по шагам), rollback/cancel/reactivate, видят все заявки.

Сотрудники

URL: /team-info/  •  Доступ: все авторизованные (управление — admin/leader)

Назначение

Карточки сотрудников команды с контактами, статусами и кастомными полями + вкладка приглашений новых участников. Статусы сотрудника: Работает (зелёный), В отпуске (жёлтый), Болеет (фиолетовый), Командировка (синий), Уволен (красный).

Как это сделать

  • Добавить сотрудника: «Добавить» → в форме справа: фото (клик по аватару → обрезка), Ф.И.О., роль, статус; поля Reddy ID, Логин CRM, контакты (рабочий/личный Telegram, почта, телефон), День рождения / Начало работы, Офис, Компания, Примечание → «Сохранить».
  • Изменить статус: клик по строке → в карточке справа dropdown статуса → выбрать один из 5 статусов.
  • Редактировать/удалить: клик по строке → карточка → «Редактировать» или «Удалить».
  • Найти: поиск «по имени, должности…».
  • Пригласить участника (admin/leader): суб-вкладка «Приглашения» → секция «Пригласить вручную» → переключатель Reddy ID / Email → ввести значение → выбрать Роль → «Отправить приглашение». Список «Ожидают подтверждения» показывает pending-инвайты и гостей.
  • Экспорт/импорт: кнопки «Экспорт списка» / «Импорт списка» в шапке (JSON).

Кто может

  • Просмотр карточек — все роли с правом view на «Сотрудники».
  • Добавление/редактирование/удаление и вкладка «Приглашения» — admin/leader (для остальных эти кнопки и суб-вкладки скрыты).

Расчёт трафика

URL: /traffic-calculation/  •  Доступ: все авторизованные роли

Назначение

Пошаговый калькулятор распределения % трафика между партнёрами на основе пополнений, качества и правил автоотключения. Состоит из 9 шагов: Партнёры → Пополнения → Качество → % Работы → Автоотключение → Нарушения → Отчёт → Настройки → Результаты.

Как это сделать

  • Выбрать партнёров (шаг 1): кнопки «Всех» / «По методу» в тулбаре → отметить партнёров для анализа.
  • Пройти расчёт: последовательно переключайте шаги в индикаторе (клик по номеру шага) — ввод пополнений, качества, % работы, правил автоотключения и нарушений.
  • Получить результат: шаги «Отчёт» / «Результаты» — сводка и экспорт.
  • Сбросить: кнопка «Сбросить всё» → подтверждение.

Кто может

Все роли с правом view на «Трафик»; редактирование данных аналитики — по матрице прав.

Excel-отчёты

URL: /excel-reports/  •  Доступ: все авторизованные роли

Назначение

Wizard-обработка xlsx-файлов по встроенным шаблонам с журналом операций и экспортом результата. Доступно 6 встроенных шаблонов: active-users, analytics-t9, btag, combined-report, deposits-withdrawals, registrations.

Как это сделать

  • Сгенерировать отчёт: пройдите шаги мастера — выберите шаблон → загрузите нужные xlsx → обработка → результат.
  • Загрузить файл: в зоне загрузки шага перетащите файл или кликните для выбора; после успешной загрузки внутри зоны появляется «✓ Загружен» и число строк.
  • Посмотреть журнал операций: плавающая кнопка справа → выдвигается панель «Журнал операций»; «Очистить» удаляет записи.
  • Сбросить/начать заново: действие сброса требует подтверждения в модальном окне.

Кто может

Все роли с правом view на «Отчёты».

Настройки

URL: /sync/  •  Доступ: все авторизованные роли

Назначение

Личный профиль пользователя + системная информация (хранилище, сессия, кеш) + выход из аккаунта.

Как это сделать

  • Заполнить профиль: карточка «Профиль» → ФИО, Логин CRM, контакты (рабочий/личный Telegram, почта, телефон), День рождения / Начало работы, Офис, Компания, Примечание → «Сохранить изменения». (Reddy ID, рабочая почта, роль — только для чтения.)
  • Переименовать команду (только leader): в шапке профиля у названия команды иконка-карандаш → клик → ввод нового имени в окне ввода.
  • Очистить локальный кеш: карточка «Система» → строка «Локальный кэш» → «Очистить» (с подтверждением).
  • Посмотреть состояние: карточка «Система» — статус облачного хранилища, «Сессия до» (время), размер локального кэша. В шапке — индикатор «Подключено».
  • Выйти: карточка «Опасная зона» → «Выйти из аккаунта» (с подтверждением; удаляет локальные данные авторизации).

Кто может

Все авторизованные — свой профиль. Переименование команды — только leader.

Вход / Авторизация

URL: /login/ (+ callback.html, waiting-invite.html)  •  Доступ: публичный (вход)

Назначение

Точка входа через Google OAuth и сценарии регистрации / ожидания доступа для новых пользователей и гостей.

Как это сделать

  • Войти: «Войти через Google» → авторизация → возврат и переход в систему.
  • Зарегистрироваться (новый пользователь): форма «Регистрация в системе» → ввести Reddy ID (4–15 цифр) → опционально галочка «Я руководитель» (тогда заполнить Название команды и описание) → «Отправить запрос».
  • После одобрения admin'ом — выбрать путь: экран «Добро пожаловать» → «Создать команду» (стать руководителем) ИЛИ «Ожидать приглашение» (руководитель пригласит).
  • Состояния доступа: «Запрос отправлен» (ждёт admin), «Запрос отклонён» (выбрать другую команду), «Доступ заблокирован» (обратиться к руководителю), «Создать хранилище» (первый вход лидера).
  • Обновить статус / выйти: на карточке доступа — кнопки «Обновить» и «Выйти из аккаунта».

Кто может

Вход доступен всем через Google-аккаунт. Дальнейший доступ определяется ролью (см. раздел «Роли и права»). Гость попадает на экран ожидания приглашения.

Как устроено (для разработчиков)

Краткая техническая справка — на втором плане относительно пользовательских сценариев выше.

Стек

Frontend — vanilla JS + Lit Web Components (CDN ESM: <app-modal>, <app-table>, <app-form-dropdown>), HTML/CSS. Backend — Google Apps Script (Web App). Авторизация — Google OAuth 2.0.

Хранилище

Данные в Google Sheets (Storage v2.0): системный файл _System/AIAdminka_Core + по командам Teams/T001_…/Data, медиа — в Google Drive. Доступ из фронтенда — через named-методы CloudStorage (не raw fetch).

Синхронизация

Incremental cache sync (v2.36): фронтенд отправляет clientTs, backend сравнивает с namespace ts и возвращает {notModified} либо delta с deletedIds. Между вкладками — SharedWorker + fallback через localStorage (Safari <15). Кеш разделён по окружениям (cs-cache-{env}-).

Безопасность

OAuth 2.0 с проверкой audience (client_id) на бэкенде; CSRF state в callback; CSP блокирует inline-скрипты; Utils.escapeHtml() + авто-эскейп Lit для защиты от XSS; роль проверяется на сервере (не в localStorage).

Дизайн-система

Токены в shared/css/variables.css: accent #fdbe2f, glassmorphism, шрифт TT Firs Neue, статус-цвета. Готовые классы: .btn-*, .card, .form-*, .status-badge, .dropdown-wrap. Уведомления — Toast вместо alert.

Решение проблем

«Permission denied» при загрузке страницы

  • Проверьте что ваша роль имеет доступ к этому модулю (см. таблицу ролей)
  • Очистите кеш браузера и перезайдите
  • Admin может проверить логи в админ-панели → Audit Log

Сессия истекла

  • Приложение автоматически переадресует на /login при истёкшем токене
  • За 10 минут до истечения показывается предупреждение с возможностью продлить

Данные не синхронизируются между вкладками

  • Проверьте что вы залогинены под одним аккаунтом в обеих вкладках
  • Safari <15 без SharedWorker — fallback через storage event (задержка до 3 секунд)
  • В DevTools Console: console.table(window.__cacheStats.byNs) — статистика cache hit-rate

Кеш «застрял» (старые данные)

  • Hard refresh: Ctrl+Shift+R
  • Очистка кеша: DevTools → Application → Local Storage → удалить cs-cache-* ключи
  • CACHE_VERSION envelope защищает от poisoning — но при сильном повреждении нужен ручной clear

Загрузка аватара / файла не работает

  • Максимальный размер: 5 МБ для аватаров
  • MIME-валидация: jpg, png, webp, gif
  • Drive должен быть доступен для записи (admin настраивает permission)
