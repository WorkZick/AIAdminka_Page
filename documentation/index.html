1. Что это?

SimpleAIAdminka — административная панель для работы с данными и командой. Современный минималистичный дизайн с единым стилем для всех модулей.

Модули:

  • Сотрудники — карточки сотрудников с настраиваемыми полями
  • Расчет трафика — управление партнерами и аналитика
  • Отчеты — обработка файлов по шаблонам
  • Партнеры — управление информацией о партнерах
  • Настройки — облачное хранение через Google Sheets
  • Главная — сетка модулей с навигацией
  • Документация — справочное руководство

Особенности:

Локальное хранение

Данные в браузере, нет серверов

Простота

Понятный интерфейс без лишнего

Адаптивность

Работает на любых устройствах

Безопасность

Данные не покидают устройство

2. Сотрудники

Новый дизайн (v2.0.0): Полностью переработанный интерфейс в современном минималистичном стиле с табличным представлением данных.

Интерфейс:

  • Боковое меню: Навигация по всем модулям системы со сворачиванием
  • Таблица сотрудников: Табличное представление с аватарами (80x80px)
  • Правая панель: Статистика команды или карточка выбранного сотрудника
  • Поиск: Быстрый поиск сотрудников в реальном времени
  • Сортировка: По любому столбцу таблицы

Работа с сотрудниками:

  • Добавление: Кнопка "+ Добавить" → форма в правой панели
  • Редактирование: Клик по сотруднику → кнопка "Редактировать"
  • Удаление: Кнопка "Удалить" с подтверждением
  • Фото: Загрузка аватара при создании/редактировании
  • Статус: Dropdown выбор статуса прямо в карточке

Поля сотрудника:

  • ФИО: Полное имя сотрудника
  • Должность: Позиция в компании
  • Грейд: Junior / Middle / Senior / Lead
  • Статус: Работает / Отпуск / Больничный / Декрет / Уволен
  • Логин УРС: Системный логин
  • Reddy: ID в Reddy
  • Корп. e-mail: Корпоративная почта
  • Корп. Telegram: Telegram username
  • Корп. телефон: Рабочий номер
  • Дата рождения: ДД/ММ/ГГГГ
  • Дата устройства: ДД/ММ/ГГГГ
  • Комментарии: Дополнительная информация

Статусы:

  • Работает: Зелёный статус (активный сотрудник)
  • Отпуск: Жёлтый статус (временно отсутствует)
  • Больничный: Красный статус (на больничном)
  • Декрет: Фиолетовый статус (декретный отпуск)
  • Уволен: Синий статус (бывший сотрудник)

Система шаблонов (NEW v2.1.0):

  • Создание шаблонов: Настройте набор полей для разных типов сотрудников
  • Типы полей: Текст, Email, Телефон, Дата, Текстовая область
  • Основной шаблон: Автоматически применяется при добавлении нового сотрудника
  • Управление: Редактирование, переименование и удаление шаблонов
  • Гибкость: Неограниченное количество настраиваемых полей
  • Применение: Выбор шаблона через селектор при добавлении сотрудника

Экспорт/импорт:

  • Экспорт: Выгрузка всего списка в JSON
  • Импорт: Загрузка JSON файла с сотрудниками
  • Сохранение фото: Аватары сохраняются в base64
  • Шаблоны: Сохраняются в localStorage отдельно

Навигация (NEW v2.1.0):

  • История навигации: Возврат к предыдущему состоянию при закрытии формы
  • Toggle карточки: Повторный клик по сотруднику закрывает карточку
  • Автосохранение: Данные сохраняются при переключении между экранами
  • Защита данных: Предупреждение при закрытии формы с несохраненными изменениями

Дизайн:

  • Цветовая схема: Тёмный фон (#2e2e2e), светлые панели (#f2f2f2)
  • Шрифт: TT Firs Neue (минимализм)
  • Акцент: Жёлтый (#fdbe2f) для активных элементов
  • Адаптивность: Работает на мобильных устройствах

3. Расчет трафика

Модуль для управления партнерами, субагентами и аналитики трафика. Включает автоматическое отслеживание статусов и гибкую систему фильтрации.

Управление партнерами:

  • Добавление: Форма с полями Метод, Субагент, Субагент ID
  • Методы: Выпадающий список с возможностью добавления/удаления методов
  • Статусы: Автоматические флаги 'новый', 'старый', 'закрыт'
  • Автообновление: Статус меняется на 'старый' через 30 дней
  • Экспорт: Выгрузка списка партнеров в JSON (только базовые данные, без аналитики)
  • Импорт: Загрузка списка партнеров из JSON с сохранением существующих данных аналитики

Импорт и экспорт партнеров (NEW):

  • Экспорт в JSON: Кнопка "Экспорт" на вкладке "Управление партнерами". Сохраняет только базовые данные: метод, субагент, ID, статус, дата
  • Импорт из JSON: Кнопка "Импорт" на вкладке "Управление партнерами". Предпросмотр файла перед импортом, умное слияние данных
  • Защита данных: Данные аналитики не экспортируются и не могут быть перезаписаны при импорте

Статусы партнеров:

  • Новый: Только что добавленный партнер (синий бейдж)
  • Старый: Партнер старше 30 дней или вручную изменен (оранжевый бейдж)
  • Закрыт: Неактивный партнер (красный бейдж)

Аналитика (6 шагов):

  • Шаг 1: Выбор партнеров для анализа (все / по методу / выборочно)
  • Шаг 2: Загрузка файлов пополнений и выводов (.xlsx)
  • Шаг 3: Загрузка отчета "Контроль качества работы"
  • Шаг 4: Ввод % времени работы на пополнения и вывод
  • Шаг 5: Загрузка отчета автоотключений
  • Шаг 6: Ввод данных по нарушениям (необязательный)

Отчет:

  • Автоформирование: После завершения аналитики
  • Статистика: Всего субагентов, новых, старых, закрытых
  • Детальный отчет: Таблица с 23 колонками
  • Экспорт в Excel: Скачивание в формате .xlsx
  • Калькулятор трафика: Оценка партнеров по настраиваемым параметрам

Калькулятор % трафика:

  • Расчет по методам: Проценты рассчитываются отдельно для каждого метода (100% на метод)
  • Настройки параметров: 16 параметров для оценки с 4 уровнями
  • Сортировка: Партнеры внутри метода отсортированы по проценту
  • Экспорт: Результаты можно выгрузить в Excel

4. Отчеты

Обработка файлов по шаблонам. Работает с .xlsx файлами.

Доступные шаблоны:

  • Депозиты/Выводы: сопоставление транзакций из разных источников
  • Объединение файлов: слияние нескольких файлов в один
  • Регистрации: обработка данных регистраций

Как использовать:

  1. Выберите шаблон
  2. Загрузите файлы (кнопка "Выбрать файлы")
  3. Нажмите "Обработать"
  4. Скачайте результат

Требования к файлам:

  • Формат: .xlsx
  • Обязательны заголовки в первой строке
  • Структура зависит от шаблона (см. логи)

5. Партнеры

Новый дизайн (v2.3.0): Полностью переработанный интерфейс в современном минималистичном стиле, аналогичный Сотрудникам.

Интерфейс:

  • Боковое меню: Навигация по всем модулям системы со сворачиванием
  • Таблица партнеров: Табличное представление с аватарами, счётчиками и статусами
  • Правая панель: Статистика или карточка выбранного партнера
  • Поиск: Быстрый поиск партнеров в реальном времени
  • Сортировка: По любому столбцу таблицы

Колонки таблицы:

  • DEP | WITH | COMP: Счётчики (настраиваются в карточке)
  • Фото: Аватар партнера
  • Ф.И.О.: Полное имя партнера
  • Метод: Способ взаимодействия
  • Субагент: Название компании
  • ID Субагента: Уникальный идентификатор
  • Статус: Текущий статус партнера

Статусы:

  • Работает: Зелёный статус (активный партнер)
  • В отпуске: Жёлтый статус (временно отсутствует)
  • Командировка: Синий статус (в поездке)
  • Уволен: Красный статус (бывший партнер)
  • Болеет: Фиолетовый статус (на больничном)

Система шаблонов:

  • Создание шаблонов: Настройте набор полей для разных типов партнеров
  • Типы полей: Текст, Email, Телефон, Дата, Текстовая область
  • Основной шаблон: Автоматически применяется при добавлении нового партнера

6. Настройки

Облачная синхронизация данных через Google Sheets с системой контроля доступа. Позволяет обмениваться данными между пользователями.

Доступ к проекту:

  • GitHub Pages: https://workzick.github.io/AIAdminka_Page/
  • Требуется: Google аккаунт для авторизации

Авторизация:

  • Войти через Google: OAuth авторизация с автоматическим получением email и имени
  • Запросить доступ: Отправка запроса администратору для новых пользователей
  • Проверить доступ: Проверка статуса запроса (защита от спама - 10 сек)
  • Выйти: Выход из аккаунта с очисткой данных сессии

Статусы подключения:

  • Не авторизован: Серый статус - требуется вход через Google
  • Ожидание: Жёлтый статус - запрос отправлен, ожидает одобрения
  • Подключено: Зелёный статус - полный доступ к функционалу

Экспорт данных:

  • Экспорт в облако: Сохранение локальных данных в Google Sheets
  • Данные: Название проекта, версия, автор, заметки
  • Автоматическое обновление: Данные перезаписываются при повторном экспорте

Импорт данных:

  • Выбор пользователя: Список доступных пользователей для импорта
  • Права доступа: Видны только данные тех, кто разрешил вам доступ
  • Предпросмотр: Просмотр данных перед импортом

Безопасность:

  • OAuth 2.0: Безопасная авторизация через Google
  • Токен сессии: Автоматическое истечение через 1 час
  • Контроль доступа: Только одобренные пользователи имеют доступ
  • Защита от спама: Ограничение частоты запросов

8. Решение проблем

Типичные проблемы:

Файлы не загружаются

Решение:

  • Проверьте формат файла (.xlsx для Excel)
  • Откройте файл в Excel, проверьте заголовки
  • Пересохраните файл

Данные не сохраняются

Решение:

  • Очистите кэш браузера (сначала экспортируйте данные!)
  • Проверьте свободное место в localStorage
  • Используйте другой браузер

Медленная работа

Решение:

  • Закройте другие вкладки
  • Разделите большие файлы на части
  • Обновите браузер

Полный сброс:

  1. Откройте инструменты разработчика (F12)
  2. Вкладка "Application" → "Local Storage"
  3. Удалите записи SimpleAIAdminka
  4. Перезагрузите страницу

Внимание!

При сбросе ВСЕ данные будут удалены. Обязательно сделайте экспорт перед сбросом.

Системные требования:

  • Chrome 80+, Firefox 75+, Safari 13+, Edge 80+
  • JavaScript включён
  • LocalStorage: 50+ МБ свободно
  • Для больших файлов: 4+ ГБ RAM

9. Архитектура

Проект полностью автономен и не требует внешних зависимостей. Все ресурсы хранятся локально.

Структура папки shared/:

  • shared/storage.js — единый модуль для работы с localStorage
  • shared/utils.js — библиотека утилит (escapeHtml, formatDate, copyToClipboard и др.)
  • shared/icons.js — локальная система иконок (замена Lucide CDN)
  • shared/icons/ — папка с 32 SVG иконками

Система иконок (v2.4.2):

Проект использует локальные SVG иконки вместо внешней библиотеки Lucide:

  • LocalIcons.createIcons() — заменяет все <i data-lucide="name"> на SVG
  • Автозагрузка: иконки загружаются автоматически при DOMContentLoaded
  • Кэширование: загруженные иконки кэшируются для производительности
  • Совместимость: API совместим с Lucide (window.lucide.createIcons)

Безопасность (CSP):

Все HTML файлы содержат Content-Security-Policy заголовки:

  • script-src 'self': только локальные скрипты
  • style-src 'self' 'unsafe-inline': локальные стили + inline
  • connect-src 'self': только локальные запросы + Google API
  • frame-ancestors 'self': защита от clickjacking
