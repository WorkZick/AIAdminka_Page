1. Что это?

SimpleAIAdminka — административная панель для работы с данными и командой. Интерфейс стилизован под Windows XP.

Модули:

  • Team Info — карточки сотрудников с настраиваемыми полями
  • Расчет трафика — управление партнерами и аналитика
  • Excel отчёты — обработка файлов по шаблонам
  • Партнеры — управление информацией о партнерах
  • Рабочий стол — drag & drop иконок, сохранение позиций
  • Документация — справочное руководство

Особенности:

💾 Локальное хранение

Данные в браузере, нет серверов

🚀 Простота

Понятный интерфейс без лишнего

📱 Адаптивность

Работает на любых устройствах

🔒 Безопасность

Данные не покидают устройство

2. Team Info

Новый дизайн (v2.0.0): Полностью переработанный интерфейс в современном минималистичном стиле с табличным представлением данных.

Интерфейс:

  • Боковое меню: Навигация по всем модулям системы со сворачиванием
  • Таблица сотрудников: Табличное представление с аватарами (80x80px)
  • Правая панель: Статистика команды или карточка выбранного сотрудника
  • Поиск: Быстрый поиск сотрудников в реальном времени
  • Сортировка: По любому столбцу таблицы

Работа с сотрудниками:

  • Добавление: Кнопка "+ Добавить" → форма в правой панели
  • Редактирование: Клик по сотруднику → кнопка "Редактировать"
  • Удаление: Кнопка "Удалить" с подтверждением
  • Фото: Загрузка аватара при создании/редактировании
  • Статус: Dropdown выбор статуса прямо в карточке

Поля сотрудника:

  • ФИО: Полное имя сотрудника
  • Должность: Позиция в компании
  • Грейд: Junior / Middle / Senior / Lead
  • Статус: Работает / Отпуск / Больничный / Декрет / Уволен
  • Логин УРС: Системный логин
  • Reddy: ID в Reddy
  • Корп. e-mail: Корпоративная почта
  • Корп. Telegram: Telegram username
  • Корп. телефон: Рабочий номер
  • Дата рождения: ДД/ММ/ГГГГ
  • Дата устройства: ДД/ММ/ГГГГ
  • Комментарии: Дополнительная информация

Статусы:

  • Работает: Зелёный статус (активный сотрудник)
  • Отпуск: Жёлтый статус (временно отсутствует)
  • Больничный: Красный статус (на больничном)
  • Декрет: Фиолетовый статус (декретный отпуск)
  • Уволен: Синий статус (бывший сотрудник)

Статистика:

  • Общее количество: Счётчик всех сотрудников
  • По статусам: Разбивка по каждому статусу
  • Автообновление: При изменении данных

Экспорт/импорт:

  • Экспорт: Выгрузка всего списка в JSON
  • Импорт: Загрузка JSON файла с сотрудниками
  • Сохранение фото: Аватары сохраняются в base64

Дизайн:

  • Цветовая схема: Тёмный фон (#2e2e2e), светлые панели (#f2f2f2)
  • Шрифт: TT Firs Neue (минимализм)
  • Акцент: Жёлтый (#fdbe2f) для активных элементов
  • Адаптивность: Работает на мобильных устройствах

3. Расчет трафика

Модуль для управления партнерами, субагентами и аналитики трафика. Включает автоматическое отслеживание статусов и гибкую систему фильтрации.

Управление партнерами:

  • Добавление: Форма с полями Метод, Субагент, Субагент ID
  • Методы: Выпадающий список с возможностью добавления/удаления методов
  • Статусы: Автоматические флаги 'новый', 'старый', 'закрыт'
  • Автообновление: Статус меняется на 'старый' через 30 дней
  • Экспорт: Выгрузка списка партнеров в JSON (только базовые данные, без аналитики)
  • Импорт: Загрузка списка партнеров из JSON с сохранением существующих данных аналитики

Импорт и экспорт партнеров (NEW):

  • Экспорт в JSON:
    • Кнопка "Экспорт" на вкладке "Управление партнерами"
    • Сохраняет только базовые данные: метод, субагент, ID, статус, дата
    • Исключает данные аналитики (back, cringe, нарушения и т.д.)
    • Файл: partners_export_ГГГГ-ММ-ДД.json
  • Импорт из JSON:
    • Кнопка "Импорт" на вкладке "Управление партнерами"
    • Предпросмотр файла перед импортом
    • Умное слияние: для существующих партнеров сохраняет данные аналитики
    • Для новых партнеров игнорирует данные аналитики из файла
    • Автоматическое обновление интерфейса после импорта
  • Защита данных:
    • Данные аналитики не экспортируются и не могут быть перезаписаны при импорте
    • Работает даже при изменении структуры данных (добавлении новых колонок)
    • Предупреждение о замене данных при импорте

Статусы партнеров:

  • Новый: Только что добавленный партнер (синий бейдж)
  • Старый: Партнер старше 30 дней или вручную изменен (оранжевый бейдж)
  • Закрыт: Неактивный партнер (красный бейдж)

Таблица партнеров:

  • Просмотр: Все партнеры в табличном виде
  • Фильтр: По методам для быстрого поиска
  • Счетчик: Общее количество субагентов с учетом фильтра
  • Действия: Редактирование и удаление партнеров

Аналитика (5 шагов):

Шаг 1: Выбор партнеров

  • Все: Выбор всех партнеров для анализа
  • По методу: Выбор одного или нескольких методов
  • Выборочно: Ручной выбор конкретных партнеров
  • Предпросмотр: Таблица выбранных партнеров
  • Сброс: Кнопка "Сбросить всё" для начала нового анализа

Шаг 2: Пополнения и выводы

  • Загрузка файлов: .xlsx файлы с данными о транзакциях
  • Автоподсчет: Back и Cringe из колонки "Доп. параметры №1"
  • Множественная загрузка: Поддержка нескольких файлов одновременно

Шаг 3: Контроль качества

  • Загрузка отчета: "Контроль качества работы" .xlsx файл
  • Автосбор данных: Количество пополнений/выводов, обращения, проценты успешности
  • Обработка: Данные автоматически привязываются к субагентам

Шаг 4: % Времени работы

  • Двухколоночный макет: Слева - форма для заполнения, справа - выбор субагента
  • Поиск субагента: Быстрый поиск по названию
  • Выбор: Список всех выбранных партнеров из шага 1 (10 видимых элементов)
  • Заполнение: Ручной ввод процентов времени работы:
    • % Времени работы на пополнения (0-100%)
    • % Времени работы на вывод (0-100%)
  • Автосохранение: Данные сохраняются при переключении между субагентами
  • Сброс данных:
    • Кнопка "Сбросить" - для текущего субагента
    • Кнопка "Сбросить всё" - для всех выбранных субагентов
  • Валидация: Кнопка "Далее" активна только при заполнении данных хотя бы для одного партнера

Шаг 5: Автоотключения

  • Загрузка файлов: "Отчет по процентовкам" .xlsx
  • Подсчет: Автоматическое определение количества автоотключений

Шаг 6: Нарушения

  • Двухколоночный макет: Слева - форма для заполнения, справа - выбор субагента
  • Поиск субагента: Быстрый поиск по названию
  • Выбор: Список всех выбранных партнеров из шага 1 (10 видимых элементов)
  • Заполнение: Ручной ввод данных по нарушениям с кнопками +/-:
    • Игнорирование чатов
    • Игнор Webmanagement
    • Очереди на пополнение
    • Очереди на вывод
    • Зачисление вне лимитов
    • Одобрение неверной суммы
    • Другие нарушения
  • Автосохранение: Данные сохраняются при переключении между субагентами
  • Сброс данных:
    • Кнопка "Сбросить" - для текущего субагента
    • Кнопка "Сбросить всё" - для всех выбранных субагентов
  • Валидация: Кнопка "Завершить" активна только при заполнении данных хотя бы для одного партнера

Навигация и редактирование (NEW):

  • Индикатор шагов: Кликабельные номера шагов для быстрого перехода
  • Завершенные шаги: Отмечаются зеленой галочкой
  • Редактирование после завершения: Возможность вернуться на любой шаг и изменить данные
  • Сохранение изменений: Все изменения автоматически применяются при повторном завершении

Отчет (ОБНОВЛЕНО):

  • Автоформирование: После нажатия "Завершить" на шаге 6, автоматически переключается на вкладку "Отчет" и формирует отчет
  • Статистика: Всего субагентов, новых, старых, закрытых
  • Детальный отчет: Полная таблица сразу на странице (без модального окна) с 23 колонками:
    • Метод, Субагент, Субагент ID, Статус, Дата добавления
    • Back, Cringe, Автоотключение
    • Количество пополнений/выводов
    • Обращения (всего и 15+ мин)
    • % успешных пополнений/выводов
    • % времени работы (пополнения и вывод)
    • Нарушения: игнорирование чатов, игнор Webmanagement
    • Очереди (пополнение и вывод)
    • Зачисление вне лимитов, одобрение неверной суммы
    • Другие нарушения
  • Экспорт в Excel:
    • Кнопка "Экспорт в Excel" вместо "Подробнее"
    • Скачивание детального отчета в формате .xlsx
    • Все 23 колонки с русскими названиями
    • Файл: detailed_report_ГГГГ-ММ-ДД.xlsx
  • Калькулятор трафика: Кнопка "Рассчитать % трафика" для оценки партнеров

Управление методами:

  • Кнопка настроек рядом с выпадающим списком
  • Добавление новых методов
  • Удаление существующих методов
  • Методы доступны во всех формах и фильтрах

Хранение данных:

  • Все данные сохраняются в localStorage браузера
  • Автоматическая проверка статусов при загрузке
  • Данные не покидают ваше устройство

4. Excel отчёты

Обработка файлов по шаблонам. Работает с .xlsx файлами.

Доступные шаблоны:

  • Депозиты/Выводы: сопоставление транзакций из разных источников
  • Объединение файлов: слияние нескольких файлов в один
  • Регистрации: обработка данных регистраций

Как использовать:

  1. Выберите шаблон
  2. Загрузите файлы (кнопка "Выбрать файлы")
  3. Нажмите "Обработать"
  4. Скачайте результат

Требования к файлам:

  • Формат: .xlsx
  • Обязательны заголовки в первой строке
  • Структура зависит от шаблона (см. логи)

5. Партнеры

Модуль для управления информацией о партнерах. Простая таблица с возможностью добавления, редактирования и удаления записей.

Основное:

  • Добавление: кнопка "Добавить партнера" → заполнить форму с тремя полями
  • Редактирование: кнопка ✏️ в строке таблицы
  • Удаление: кнопка 🗑️ с подтверждением

Поля партнера:

  • Метод: Способ взаимодействия (например: WebMoney, Qiwi, Банковская карта)
  • Субагент: Название компании или организации-партнера
  • ID Субагента: Уникальный идентификатор партнера в системе

Интерфейс:

  • Таблица: Компактное отображение всех партнеров
  • Пустое состояние: Кнопка быстрого добавления первого партнера
  • Модальное окно: Удобная форма для ввода данных

Хранение данных:

  • Все данные сохраняются в localStorage браузера
  • Данные не покидают ваше устройство
  • Автоматическое сохранение при каждом изменении

7. Решение проблем

Типичные проблемы:

❌ Файлы не загружаются

Решение:

  • Проверьте формат файла (.xlsx для Excel)
  • Откройте файл в Excel, проверьте заголовки
  • Пересохраните файл

❌ Данные не сохраняются

Решение:

  • Очистите кэш браузера (сначала экспортируйте данные!)
  • Проверьте свободное место в localStorage
  • Используйте другой браузер

❌ Медленная работа

Решение:

  • Закройте другие вкладки
  • Разделите большие файлы на части
  • Обновите браузер

❌ Drag & drop не работает

Решение:

  • Обновите браузер
  • Проверьте, что JavaScript включён
  • На мобильных используйте кнопки редактирования

Полный сброс:

  1. Откройте инструменты разработчика (F12)
  2. Вкладка "Application" → "Local Storage"
  3. Удалите записи SimpleAIAdminka
  4. Перезагрузите страницу

⚠️ Внимание!

При сбросе ВСЕ данные будут удалены. Обязательно сделайте экспорт перед сбросом.

Системные требования:

  • Chrome 80+, Firefox 75+, Safari 13+, Edge 80+
  • JavaScript включён
  • LocalStorage: 50+ МБ свободно
  • Для больших файлов: 4+ ГБ RAM
